# 银行证券风险智能评估系统 — 项目实施步骤

## 项目概述

纯前端静态网页项目，将 Excel 数据转为 JSON 嵌入页面，通过 Vercel 部署。主要面向手机端使用。

**技术栈：** React + Tailwind CSS（单页应用）  
**数据方案：** Excel → JSON，嵌入前端，客户端完成所有查询/筛选  
**部署方式：** Vercel 静态部署

---

## 数据字段说明

```
用户, 身份证号, 性别, 姓名, 出身年份, 学历, 年收入（万元）, 工作领域, 工作机构, 账户状态, 账户金额
```

---

## Step 1：项目初始化

**目标：** 搭建可运行的空白 React 项目骨架。

**任务：**
- 使用 Vite + React + TypeScript 初始化项目
- 安装 Tailwind CSS 并完成配置
- 确保 `npm run dev` 可以正常启动，页面显示 "Hello World"
- 配置移动端 viewport meta 标签

**验收标准：** 本地 dev server 启动后，手机或浏览器模拟器中可以看到页面。

---

## Step 2：Excel 数据转 JSON

**目标：** 将 Excel 文件转换为项目可用的 JSON 数据文件。

Excel文件的相对路径：./personal_data.xlsx

**任务：**
- 编写一个 Python 脚本（或 Node 脚本），读取 Excel 文件，输出为 `data.json`
- JSON 格式示例：
  ```json
  [
    {
      "用户": "U001",
      "身份证号": "110101196501011234",
      "性别": "男",
      "姓名": "张三",
      "出生年份": 1965,
      "学历": "本科",
      "年收入": 15.5,
      "工作领域": "金融",
      "工作机构": "XX银行",
      "账户状态": "正常",
      "账户金额": 50000
    }
  ]
  ```
- 将生成的 `data.json` 放入项目的 `src/data/` 目录
- 在 React 中验证可以 `import data from './data/data.json'` 并 console.log 出数据

**验收标准：** 控制台能打印出完整的 JSON 数组，条数与 Excel 行数一致。

---

## Step 3：主页界面

**目标：** 完成银行风格的主页 UI，手机端适配。

**任务：**
- 顶部区域：正式标题「银行证券风险智能评估系统」
- Hero 区域：银行/金融风格图形装饰（可用 SVG/CSS 渐变实现，不依赖外部图片）
- 配色方案：深蓝 + 金色为主调，参考银行类 APP 风格
- 下方放置 3 个功能按钮（先占位，点击暂无功能）：
  1. 📋 查看用户
  2. 🔍 开启风险评估
  3. 🔎 查询数据
- 整体布局适配手机端（max-width 约 430px 居中，或 100% 宽度响应式）

**验收标准：** 手机浏览器中看到完整的主页，3 个按钮可点击（暂无跳转）。

---

## Step 4：页面路由 / 视图切换

**目标：** 实现主页与 3 个功能页面之间的切换。

**任务：**
- 可以使用 React Router，也可以用简单的 state 切换（推荐后者，更轻量）
- 每个功能页面顶部有「返回主页」按钮
- 3 个功能页面暂时只显示标题占位文字：
  - "查看用户" 页
  - "风险评估" 页
  - "查询数据" 页

**验收标准：** 点击主页按钮可进入对应页面，点击返回可回到主页。

---

## Step 5：「查看用户」功能

**目标：** 实现用户数据的分页浏览。

**任务：**
- 加载 JSON 数据，以卡片或表格形式展示用户信息
- 手机端建议使用卡片式布局（每条数据一张卡片），而非宽表格
- 实现分页功能：每页显示 10 条，底部有「上一页 / 下一页」及页码显示
- 显示总数据条数

**验收标准：** 可以翻页浏览所有数据，数据内容与 Excel 一致。

---

## Step 6：「开启风险评估」功能 — 数据筛选与展示

**目标：** 筛选出符合年龄条件的风险用户并展示。

**任务：**
- 筛选逻辑：根据当前年份（2026）和出生年份计算年龄
  - 女性 > 55 岁（即出生年份 < 1971）
  - 男性 > 60 岁（即出生年份 < 1966）
- 将筛选结果以卡片列表展示（同样支持分页）
- 顶部显示风险用户总数

**验收标准：** 筛选结果正确，手动抽查几条数据验证年龄和性别条件。

---

## Step 7：「开启风险评估」功能 — 身份证搜索

**目标：** 在风险评估页面中增加搜索功能。

**任务：**
- 页面顶部增加搜索框，placeholder 提示「输入身份证号查询」
- 输入身份证号后点击搜索（或回车触发）：
  - 如果在风险列表中找到：高亮展示该用户的完整信息卡片，标注「⚠️ 该用户在风险名单中」
  - 如果未找到：显示提示「✅ 未在风险名单中找到该用户」
- 搜索框旁有清除/重置按钮，点击后恢复完整列表

**验收标准：** 输入已知的风险用户身份证号能正确展示，输入不在名单中的身份证号显示未找到。

---

## Step 8：「查询数据」功能

**目标：** 实现全局用户数据查询。

**任务：**
- 页面有一个搜索框，支持通过「身份证号」或「姓名」查询
- 自动判断输入类型（纯数字视为身份证号，否则视为姓名）
- 查询结果：
  - 精确匹配身份证号：展示该用户完整信息
  - 姓名查询：支持模糊匹配（包含即可），可能返回多条结果
  - 无结果：显示「未找到匹配的用户」
- 结果以卡片形式展示

**验收标准：** 身份证号精确查询和姓名模糊查询均正常工作。

---

## Step 9：UI 打磨与细节优化

**目标：** 提升整体视觉效果和使用体验。

**任务：**
- 统一配色、字体、间距，确保银行系统风格一致
- 卡片信息展示优化：关键字段（姓名、身份证号）突出显示，次要字段（学历、工作机构等）适当弱化
- 按钮增加 loading 或过渡动画
- 搜索交互优化：搜索时有加载提示，空输入时给出提醒
- 风险评估页面，风险用户卡片用醒目的红色/橙色边框标识
- 确保所有页面在不同手机尺寸下显示正常

**验收标准：** 整体视觉协调，交互流畅，无明显样式问题。

---

## Step 10：构建与 Vercel 部署

**目标：** 将项目打包并部署上线。

**任务：**
- 运行 `npm run build`，确保构建无报错
- 在项目根目录创建 `vercel.json`（如需要 SPA 路由回退配置）：
  ```json
  {
    "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
  }
  ```
- 将项目推送到 GitHub 仓库
- 在 Vercel 中导入该仓库，完成部署
- 用手机访问 Vercel 分配的域名，测试所有功能

**验收标准：** 手机端通过 URL 可正常访问，3 个功能模块全部可用。
